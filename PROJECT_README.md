## Концепція

**BIBI Cars** — це повний цикл «пригону» вживаних авто з аукціонів США та Південної Кореї
до Болгарії: від ставки на аукціоні до видачі ключів клієнту. Платформа поєднує:

1. **Публічну вітрину (storefront)** — каталог авто, пошук за VIN/номером лота,
   калькулятор вартості імпорту, блог, сторінки про компанію та контакти.
2. **Внутрішню CRM** — наскрізне ведення угоди: лід → депозит → ставка/виграш →
   контракт → оплата → доставка (океан + суша) → митниця → видача.
3. **Клієнтський кабінет** — клієнт у реальному часі бачить статус свого авто,
   підписує контракти, оплачує інвойси та відстежує доставку.

Дані про авто збираються парсерами з зовнішніх джерел (bidmotors, lemon, westmotors
тощо), збагачуються історією VIN та трекінгом контейнерів (VesselFinder).

## Що вміє платформа

**Публічна частина**
- Каталог авто з фільтрами (марка, модель, рік) та живим пошуком за VIN/лотом.
- Калькулятор повної вартості імпорту (аукціонна ціна + доставка + митниця + комісія).
- Перевірка історії авто (VIN-сервіс), блог, SEO-сторінки, мультимовність (EN/BG).

**CRM / операційне ядро**
- Управління лідами (джерела, пріоритезація, SLA, нагадування, ескалації).
- Пайплайн угод (deal pipeline), депозити, інвойси, платежі.
- Юридичний воркфлоу: шаблони документів, контракти, e-підпис.
- Доставка 360: трекінг суден/контейнерів, події відвантаження, статуси.
- Фінанси: розбивка вартості, звітність, KPI, прогнозування.
- Дзвінки (Ringostat-інтеграція готова до підключення) та евристичний скоринг дзвінків.
- Сповіщення (in-app, e-mail outbox), аудит дій, управління персоналом.
- Парсери-воркери, що працюють у фоні, та движок резолвінгу/злиття даних авто.

## Ролі та кабінети

У системі рівно **чотири ролі**: `admin`, `team_lead`, `manager`, `customer`.

| Роль | Маршрут | Призначення |
|------|---------|-------------|
| **Admin** (адмін) | `/admin` | Повний контроль: вся CRM, фінанси, налаштування, парсери, персонал, KPI. |
| **Team Lead** (тімлід) | `/team/dashboard` | Дашборд команди, навантаження менеджерів, переназначення лідів, нагляд за SLA. |
| **Manager** (менеджер) | `/manager` | Особистий простір: гарячі ліди, задачі, дзвінки, інвойси, замовлення, відвантаження. |
| **Customer** (клієнт/юзер) | `/cabinet/:id` | Особистий кабінет: статус авто, контракти, інвойси, доставка, обране, порівняння. |

Єдина сторінка входу для всіх — `/cabinet/login` (вона сама визначає, staff це чи клієнт).

**Двофакторна автентифікація:**
- `admin` — TOTP (Google Authenticator), якщо увімкнено в налаштуваннях.
- `team_lead` — одноразовий код на e-mail (Email-OTP).
- `manager` — лише логін/пароль.
- `customer` — логін/пароль або Google OAuth (опційно).

## Технологічний стек

**Backend**
- Python 3.11, **FastAPI**, Uvicorn, async (Motor / **MongoDB**).
- JWT-автентифікація (`python-jose`), `bcrypt`/`passlib`, `pyotp` (TOTP).
- Фонові воркери (власний `worker_registry`), парсери (`httpx`, `beautifulsoup4`, `lxml`, `playwright`).
- PDF-генерація (`weasyprint`), `slowapi` (rate limiting), `prometheus_client` (метрики).
- Платежі: `stripe` (опційно).

**Frontend**
- **React 19**, CRACO, **Tailwind CSS** + shadcn/ui (Radix UI).
- `react-router-dom` 7, `@tanstack/react-query`, `axios`, `socket.io-client`.
- `recharts` (графіки), `leaflet` (карти), `tiptap` (редактор), `framer-motion` (анімації).
- Мультимовність: **EN / BG / UK** (укр. — лише для staff-кабінетів).

**Сховище**
- MongoDB (основна БД), файлове/обʼєктне сховище для документів та зображень.

## Швидкий старт (локально)

> Потрібні: Python 3.11, Node 18+, Yarn, MongoDB.

```bash
# 1. Backend
cd backend
cp .env.example .env                 # відредагуйте MONGO_URL / DB_NAME / JWT_SECRET
pip install -r requirements.txt
uvicorn server:fastapi_app --host 0.0.0.0 --port 8001

# 2. Frontend (в окремому терміналі)
cd frontend
cp .env.example .env                 # вкажіть REACT_APP_BACKEND_URL
yarn install
yarn start                           # http://localhost:3000
```

Повний гайд із продакшн-розгортання — у [`docs/DEPLOYMENT.md`](docs/DEPLOYMENT.md).

## Облікові записи за замовчуванням

При першому старті backend автоматично створює staff-акаунти (їх можна перевизначити
через змінні оточення — див. `backend/.env.example`):

| Роль | E-mail | Примітка |
|------|--------|----------|
| Admin | `admin@bibi.cars` | пароль задається у сидері / env |
| Manager | `manager@bibi.cars` | пароль задається у сидері / env |
| Team Lead | `teamlead@bibi.cars` | вхід через Email-OTP |

> **Безпека:** обовʼязково змініть паролі через змінні `BIBI_*_PASSWORD` перед
> продакшеном. Клієнтський тестовий обхід (`test@customer.com`) слід прибрати —
> див. `docs/DEPLOYMENT.md`.
